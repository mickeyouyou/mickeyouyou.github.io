如何通过Docker镜像提交image？

用户在仿真平台上传和运行image需要先通过本地Docker镜像上传完整的image。Docker镜像上传image的步骤如下：
A．启动Docker指定仓库地址
sudo service docker stop
sudo service docker start –insecure-registry=apollo-docker.baidu.com
（如果发现invalid option错误，需要在/etc/default/docker文件中增加一行DOCKER_OPTS=“–insecure-registry apollo-docker.baidu.com”）
B．登录仓库
sudo docker login apollo-docker.baidu.com
C．输入用户名和密码（用户名和密码请在『任务创建』页的『获取验证码』查询）
username: {USERNAME}
password: {PASSWORD}
D．拉取仿真平台的标准simulation镜像
sudo docker pull apollo-docker.baidu.com/public/demo/simulation:v0.0.1
E．修改simulation镜像tag名。若用户修改的tag名总是相同，将直接替换线上原有的同名tag。
sudo docker tag apollo-docker.baidu.com/public/demo/simulation:v0.0.1 apollo-docker.baidu.com/{USERNAME}/demo/simulation:v0.0.1
F．进入simulation镜像，在标准镜像的docker container中放置自己需要替换的算法模块
sudo docker run -it –privileged=true –entrypoint /bin/bash apollo-docker.baidu.com/{USERNAME}/demo/simulation:v0.0.1
在/apollo目录下放置sim_op目录，sim_op下面包含metainfo文件和模块启停脚本
修改镜像内容完成后exit退出镜像，Docker将返回新镜像id
# container_id
G. 基于docker container构建新的image。
sudo docker commit -m ‘comments’ container_id apollo-docker.baidu.com/{USERNAME}/demo/simulation:v0.0.1
H．推送新镜像至Apollo仿真平台的docker registry，即可在仿真平台的『image上传选择』界面更新到本次提交的image。
sudo docker push apollo-docker.baidu.com/{USERNAME}/demo/simulation:v0.0.1
